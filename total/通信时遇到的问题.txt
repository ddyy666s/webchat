语音/视频通话功能调试总结
一、遇到的问题及解决方案
问题1：通话信令发给了自己（toUserId=自己）
现象：后端日志显示 无效的通话目标用户: from=7, to=7

原因：CommunicationBar.vue 中 startVoiceCall 没有正确传递对方用户ID，导致 toUserId 为 null 或当前用户自己的ID。

解决方案：

CommunicationBar.vue 中 startVoiceCall 通过 props.currentChatUserId 获取当前选中的好友ID，并通过 emit 传递给父组件

ChatWindow.vue 中接收 startVoiceCall 事件，获取 toUserId 参数，并打开 CallDialog 弹窗

CallDialog.vue 中使用 props.targetUser.id 作为信令目标，并添加 getTargetUserId() 兼容 id 和 userId 两种格式

问题2：后端转发信令时 toUserId 为 null
现象：后端日志 无效的通话目标用户: from=7, to=null

原因：前端发送的信令中 toUserId 字段为 null，没有正确传递

解决方案：

在 CallDialog.vue 中添加 getTargetUserId() 函数，兼容 targetUser.id 和 targetUser.userId

在所有发送信令的地方（offer、answer、ice-candidate、hangup）都先获取 targetId 并检查是否为 null

问题3：对方收不到来电弹窗
现象：发起通话后，对方没有任何反应

原因：ChatWindow.vue 中没有正确监听 WebSocket 的 call 信令，或者没有正确处理 offer 消息

解决方案：

确保 websocket.ts 中 onCallSignal 回调已注册，并在 onmessage 中处理 type === 'call'

在 ChatWindow.vue 的 onCallSignal 中，当收到 action === 'offer' 且 fromUserId !== currentUserId 时，弹出 incomingCallVisible 弹窗

被叫方的 CallDialog 使用 :is-caller="false" 标记，显示"接听"按钮

问题4：WebRTC 连接建立失败
现象：通话弹窗出现了，但一直显示"正在连接..."，无法建立通话

原因：

ICE 服务器配置问题

信令交换不完整（offer/answer/ice-candidate 没有正确转发）

解决方案：

配置正确的 STUN 服务器：iceServers: [{ urls: 'stun:stun.l.google.com:19302' }]

确保后端正确转发所有信令（offer、answer、ice-candidate）

确保 peerConnection.onicecandidate 正确发送 ICE 候选

问题5：麦克风/摄像头权限被拒绝
现象：控制台报错 NotAllowedError 或 NotFoundError

原因：浏览器阻止了媒体设备访问

解决方案：

在 startCall 和 acceptCall 中添加 navigator.mediaDevices.getUserMedia 权限请求

添加详细的错误处理，区分 NotAllowedError、NotFoundError 等

在 App.vue 中添加全局"解锁音频"机制，用户首次点击页面时请求权限

提示用户手动在浏览器地址栏设置麦克风/摄像头权限为"允许"

二、最终正确的代码结构
1. CommunicationBar.vue（发起通话）
typescript
const props = defineProps<{ currentChatUserId?: number }>()
const emit = defineEmits(['startVoiceCall', 'startVideoCall'])

const startVoiceCall = () => {
  if (props.currentChatUserId) {
    emit('startVoiceCall', props.currentChatUserId)
  } else {
    ElMessage.warning('请先选择聊天对象')
  }
}
2. ChatWindow.vue（接收事件，打开弹窗，监听信令）
typescript
// 主动发起
const startVoiceCall = (toUserId: number) => {
  if (!toUserId || toUserId === currentUserId) {
    ElMessage.error('不能给自己打电话')
    return
  }
  voiceCallVisible.value = true
}

// 被动接听
const onCallSignal = (data: any) => {
  if (data.action === 'offer' && data.fromUserId !== currentUserId) {
    incomingCaller.value = { id: data.fromUserId, nickname: data.fromUserNickname }
    incomingCallType.value = data.callType
    incomingCallVisible.value = true
  }
}
3. CallDialog.vue（WebRTC 核心）
typescript
const getTargetUserId = () => props.targetUser?.id || props.targetUser?.userId

const startCall = async () => {
  const targetId = getTargetUserId()
  if (!targetId) { /* 错误处理 */ return }
  
  localStream = await navigator.mediaDevices.getUserMedia({ audio: true, video: props.callType === 'video' })
  await createPeerConnection()
  const offer = await peerConnection!.createOffer()
  await peerConnection!.setLocalDescription(offer)
  websocketService.sendCallSignal({
    action: 'offer',
    toUserId: targetId,
    callType: props.callType,
    sdp: offer.sdp
  })
}
4. websocket.ts（信令转发）
typescript
private callSignalCallbacks: MessageCallback[] = []

sendCallSignal(data: any) {
  this.ws?.send(JSON.stringify({ type: 'call', ...data }))
}

onCallSignal(callback: MessageCallback) {
  this.callSignalCallbacks.push(callback)
}

// onmessage 中
} else if (data.type === 'call') {
  this.callSignalCallbacks.forEach(cb => cb(data))
}
三、关键经验总结
问题类型	排查方法	解决方案
信令没发出去	后端日志查看 toUserId	检查前端传递的用户ID是否正确
对方没收到	检查对方 WebSocket 是否连接	确保 onCallSignal 正确注册
连接失败	检查 ICE 候选交换	配置 STUN 服务器
权限问题	浏览器控制台查看错误类型	添加权限请求和错误提示
